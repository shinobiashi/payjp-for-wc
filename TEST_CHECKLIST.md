# PAY.JP for WooCommerce — テストチェックリスト

テストモードで実施すべき検証項目の一覧です。

---

## テスト環境情報

### テスト用カード番号

| 番号 | ブランド | 期待結果 |
|------|---------|---------|
| `4242 4242 4242 4242` | Visa | 決済成功 |
| `4000 0000 0000 0002` | Visa | カード拒否（declined） |
| `4000 0027 6000 3184` | Visa | 3DS 認証が必要 |

- 有効期限: 任意の未来日（例: `12/30`）
- セキュリティコード: 任意の3桁（例: `123`）

### テスト用 PayPay アカウント（電話番号）

`080-1111-5912` 〜 `080-1111-5921`（10アカウント）

- テスト上限: **¥100 / 回**
- テスト後は全額返金すること

## 1. 事前準備・管理画面

- [ ] テスト用シークレットキー・公開鍵を管理画面（PAY.JP 設定）に入力・保存できる
- [ ] テストモードを ON にすると管理画面の PAY.JP 設定ページに「Test mode is active.」通知が表示される（チェックアウトには表示されない）
- [ ] テストモードを OFF にしても設定値が保持されている
- [ ] Webhook URL が設定画面に表示される（`/wp-json/payjp/v2/webhook`）
- [ ] PAY.JP ダッシュボードへのリンクボタンが表示される
- [ ] テストモード中にテストカード・PayPay テストアカウントへのリンクが管理画面に表示される
- [ ] クレジットカードゲートウェイを無効にするとチェックアウトに表示されない
- [ ] PayPay ゲートウェイを無効にするとチェックアウトに表示されない

---

## 2. クレジットカード決済 — Classic Checkout（即時課金）

### 2-1. ゲスト購入

- [ ] 支払い方法欄に PAY.JP カード入力 Widget が表示される
- [ ] テストカード（成功）を入力して注文完了できる
- [ ] 注文画面に `_payjp_payment_flow_id`（pflw_xxx）が保存されている
- [ ] WooCommerce 注文ステータスが「処理中」になる

### 2-2. ログイン済みユーザー（カード保存しない）

- [ ] 新規カードで決済できる
- [ ] 「このカードを保存する」チェックをオフにして決済 → マイアカウントにカードが保存されない

### 2-3. ログイン済みユーザー（カード保存あり）

- [ ] 「このカードを保存する」チェックをオンにして決済完了できる
- [ ] マイアカウント > 支払い方法に保存されたカードが表示される
- [ ] 2回目の購入時に保存カードがリストに表示される
- [ ] 保存カードを選択して決済できる（新たな Widget 入力なし）

### 2-4. エラーケース

- [ ] 拒否カード（`4000 0000 0000 0002`）で適切なエラーメッセージが表示される
- [ ] カード番号未入力で注文しようとするとエラーになる

---

## 3. クレジットカード決済 — Classic Checkout（手動キャプチャ）

> 設定: クレジットカードゲートウェイ > 「支払いキャプチャ」を「与信確保のみ」に変更

- [ ] 決済完了後の注文ステータスが「保留中」になる（即時課金ではない）
- [ ] 注文詳細の `_payjp_capture_method` メタが `manual` になっている
- [ ] 注文ステータスを「完了」に変更するとキャプチャが実行される
- [ ] 「PAY.JP: Payment captured successfully.」という注文メモが追加される
- [ ] PAY.JP ダッシュボードで対象 Payment Flow が captured になっている
- [ ] すでに即時課金した注文を「完了」にしても二重課金されない

---

## 4. クレジットカード決済 — Block Checkout（即時課金）

- [ ] Block Checkout（ブロックエディター版チェックアウト）でカード入力 Widget が表示される
- [ ] ゲストとして新規カードで決済できる
- [ ] ログイン済みユーザーが新規カードで決済できる（保存あり・なし両方）
- [ ] 保存済みカードのリストが表示され選択できる
- [ ] 保存カードを選択して決済できる
- [ ] 支払い方法が `payjp_card`（PayPay ではなく）で記録されている
- [ ] 保存カードがある状態で PayPay を選択しても `payjp_card` に戻らない

---

## 5. クレジットカード 返金

- [ ] 注文詳細から全額返金を実行できる
- [ ] 全額返金後に注文ステータスが「返金済み」になる
- [ ] 部分返金（例: 合計の半額）を実行できる
- [ ] 同一注文に対して2回目の部分返金ができる（複数回部分返金）
- [ ] 返金後に PAY.JP ダッシュボードで返金が確認できる
- [ ] 返金済みの注文に対してさらに返金しようとすると適切なエラーになる

---

## 6. PayPay 決済 — 即時課金

- [ ] チェックアウトに PayPay 支払いオプションが表示される
- [ ] 「注文する」を押すと PAY.JP の PayPay 画面（QR またはディープリンク）にリダイレクトされる
- [ ] テスト用 PayPay アカウントで承認すると注文完了ページに戻る
- [ ] 注文ステータスが「処理中」になる
- [ ] 注文メタの `_payjp_payment_method` が `paypay` になっている
- [ ] ユーザーが PayPay 画面でキャンセルして戻ったとき、カートが残っており適切なメッセージが表示される

---

## 7. PayPay 決済 — 手動キャプチャ

> 設定: PayPay ゲートウェイ > 「支払いキャプチャ」を「与信確保のみ」に変更

- [ ] PayPay 決済完了後の注文ステータスが「保留中」になる
- [ ] `_payjp_capture_method` メタが `manual` になっている
- [ ] 注文ステータスを「完了」に変更するとキャプチャが実行される
- [ ] 「PAY.JP PayPay: Payment captured successfully.」という注文メモが追加される

---

## 8. PayPay 返金

- [ ] 全額返金を実行できる
- [ ] 部分返金を実行できる
- [ ] 返金後に PAY.JP ダッシュボードで返金が確認できる

---

## 9. カード保存（マイアカウント）

> マイアカウント > 支払い方法

- [ ] 「新しい支払い方法を追加」から PAY.JP Setup Flow が起動する
- [ ] テストカードを入力して保存が完了する
- [ ] マイアカウントの支払い方法一覧にカード（マスク番号・ブランド・有効期限）が表示される
- [ ] 保存カードを削除すると PAY.JP ダッシュボードでも detach される
- [ ] 削除後にマイアカウントのリストから消えている

---

## 10. Webhook 処理

> PAY.JP ダッシュボード > Webhook > テスト送信 を使用

### 10-1. 正常系

- [ ] `payment_flow.succeeded` イベントで対象注文が「処理中」になる
- [ ] `payment_flow.succeeded` で注文メモ「PAY.JP: Payment confirmed via webhook.」が追加される
- [ ] `payment_flow.payment_failed` イベントで対象注文が「失敗」になる
- [ ] `payment_flow.payment_failed` で注文メモ「PAY.JP: Payment failed via webhook.」が追加される
- [ ] `refund.created` イベントで注文メモ「PAY.JP: Refund ...」が追加される
- [ ] 管理画面から手動で返金した場合、`refund.created` Webhook で注文メモが**重複しない**（冪等性）

### 10-2. セキュリティ

- [ ] Webhook シークレットが未設定の場合でもリクエストを受け付ける（シークレット未設定時のフォールバック）
- [ ] Webhook シークレットを設定後、不正なシグネチャのリクエストが `401` で拒否される
- [ ] 正しいシグネチャのリクエストが正常に処理される

---

## 11. WooCommerce Subscriptions（サブスクリプション対応）

> WooCommerce Subscriptions プラグインが必要

- [ ] サブスクリプション商品をカードで購入できる（初回決済）
- [ ] 初回決済後にカードが保存され、マイアカウントに表示される
- [ ] 次の更新日を手動トリガーすると自動更新決済が実行される
- [ ] 自動更新の注文が「処理中」になる
- [ ] $0 のサブスクリプション（無料トライアル更新）が API 呼び出しなしで完了する
- [ ] 保存カードなしで更新が来た場合、注文が「失敗」になり管理者に通知される
- [ ] マイアカウントからサブスクリプションの支払い方法を変更できる

---

## 12. アンインストール

- [ ] プラグインを削除すると `payjp_settings` オプションが wp_options から削除される
- [ ] `woocommerce_payjp_card_settings` が削除される
- [ ] `woocommerce_payjp_paypay_settings` が削除される
- [ ] ユーザーメタ `_payjp_customer_id` が全ユーザーから削除される
- [ ] 注文メタ（`_payjp_payment_flow_id` 等）は**残っている**（決済記録として保持）

---

## 13. マルチサイト（該当する場合）

- [ ] サブサイトごとに独立した API キーを設定できる
- [ ] あるサブサイトの設定変更が他サブサイトに影響しない
- [ ] ネットワーク管理からプラグインを削除すると全サブサイトのオプションが削除される
