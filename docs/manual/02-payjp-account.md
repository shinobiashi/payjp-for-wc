# 第2章 PAY.JP アカウントの作成

決済を受け付けるには、まず PAY.JP のアカウントが必要です。アカウントの作成は**無料**で、登録した直後から「テスト環境」（お金が動かない練習環境）を使えます。

## 2-1. アカウントを登録する

1. ブラウザで [https://pay.jp/](https://pay.jp/) を開きます。
2. 画面の「**アカウント登録**」（または「無料で始める」）ボタンをクリックします。
3. メールアドレスとパスワードを入力して登録します。
4. 登録したメールアドレスに確認メールが届くので、メール内のリンクをクリックして登録を完了します。

> [!TIP]
> パスワードは他のサービスと使い回さず、忘れないように安全な場所に控えておいてください。

## 2-2. ダッシュボードにログインする

登録が完了したら、[https://pay.jp/](https://pay.jp/) の「ログイン」からダッシュボード（PAY.JP の管理画面）にログインします。

ダッシュボードでは次のことができます。

- API バージョンの変更
- 決済の履歴・売上の確認
- **API キーの確認**（次の節で説明します。プラグインの設定に必要です）
- Webhook の設定（第4章で使います）
- 本番環境の利用申請（第6章で使います）

> [!TIP]
> このプラグインは API v2 を利用しています。そのため、ログイン後すぐに API バージョンを v2 に切り替えてください。

## 2-3. テスト用 API キーを確認する

API キーは、あなたのサイトと PAY.JP をつなぐための「合言葉」です。プラグインの設定画面（第4章）で入力するので、場所を覚えておきましょう。

1. PAY.JP ダッシュボードにログインします。
2. メニューから「**API**」（API キー）のページを開きます。
3. **テスト環境**のキーとして、次の 2 つが表示されています。
   - **テスト公開鍵** … `pk_test_` から始まる文字列
   - **テスト秘密鍵** … `sk_test_` から始まる文字列

> [!WARNING]
> **秘密鍵（`sk_` から始まるキー）は絶対に他人に教えないでください。**
> メールやチャットで送ったり、ブログ等に貼り付けたりしてはいけません。秘密鍵が漏れると、第三者があなたのアカウントで決済操作をできてしまう恐れがあります。

> [!NOTE]
> この時点では**本番用のキー（`pk_live_` / `sk_live_`）はまだ使えません**。本番用キーは、第6章で説明する「本番環境の利用申請」の審査に通ると使えるようになります。まずはテスト環境で設定と動作確認を進めましょう。
