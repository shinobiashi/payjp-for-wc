# 第5章 テスト決済をしてみよう

設定が終わったら、実際に商品を購入する流れを試してみましょう。**テストモードでは実際のお金は一切動きません**ので、安心して何度でも試せます。

## 5-1. テスト購入の流れ

1. ショップのページ（お客様が見る画面）を開きます。ログイン中の管理者のままでも、別のブラウザやシークレットウィンドウでも構いません。
2. 適当な商品をカートに入れ、チェックアウト（購入手続き）画面へ進みます。
3. 支払い方法に「**クレジットカード**」と「**PayPay**」が表示されていることを確認します。

> [!TIP]
> 支払い方法が表示されない場合は、[第8章 よくある質問](08-faq.md) の「チェックアウトに決済手段が表示されない」を確認してください。

## 5-2. テスト用クレジットカードで支払う

テストモードでは、本物のカードの代わりに**テスト専用のカード番号**を使います。
詳細は [開発ツール テストカード](https://docs.pay.jp/v2/guide/developers/testcards)を確認してください。

| 用途 | カード番号 |
|------|-----------|
| Visa（成功する） | `4242 4242 4242 4242` |
| Mastercard（成功する） | `5555 5555 5555 4444` |
| JCB（成功する） | `3530 1113 3330 0000` |
| わざと失敗させる | `4000 0000 0000 0002` |

- **有効期限** … 未来の年月なら何でも OK（例: 12/30）
- **セキュリティコード（CVC）** … 任意の 3 桁の数字で OK（例: 123）
- **名義** … 任意の名前で OK

手順:

1. 支払い方法で「クレジットカード」を選びます。
2. カード入力欄に上記のテストカード番号・有効期限・セキュリティコードを入力します。
3. 「注文する」ボタンをクリックします。
4. 3D セキュア（本人認証）の確認画面が表示された場合は、テスト用の画面で「成功」を選択します。
5. 「注文ありがとうございます」の画面が表示されれば成功です。

その後、管理画面の「WooCommerce」→「注文」を開き、注文が作成されてステータスが「**処理中**」になっていることを確認してください。

## 5-3. PayPay のテスト決済

PayPay のテストには、PAY.JP が用意している**テスト専用アカウント**を使います。自分の本物の PayPay アカウントは使いません。
詳細は [PayPay 決済について テストアカウントについて](https://docs.pay.jp/v2/guide/payments/methods/paypay#%E3%83%86%E3%82%B9%E3%83%88%E3%82%A2%E3%82%AB%E3%82%A6%E3%83%B3%E3%83%88%E3%81%AB%E3%81%A4%E3%81%84%E3%81%A6)を確認してください。

| 項目 | 値 |
|------|-----|
| テスト用電話番号 | `080-1111-5912` 〜 `080-1111-5921` のいずれか |
| パスワード | `Pay2test` |
| SMS 認証コード | `1234`（固定） |

> [!CAUTION]
> PayPay のテスト決済は **1 回 100 円まで**という上限があります。テストには 100 円以下の商品を使ってください。また、**テストが終わったら必ず全額返金してください**（返金の方法は第7章で説明します）。

手順:

1. 100 円以下のテスト商品を作成してカートに入れます（商品は後で削除して構いません）。
2. 支払い方法で「PayPay」を選び、「注文する」をクリックします。
3. PayPay の支払い画面が表示されます。QR コードが表示された場合は、「**QR コードをスキャンできない場合はこちら**」のリンクをクリックすると、スマートフォンがなくてもブラウザ上でログインして支払いを進められます。
4. 上記のテスト用電話番号とパスワードでログインし、SMS 認証コード `1234` を入力して支払いを完了します。
5. ショップの「注文ありがとうございます」画面に戻れば成功です。

> [!NOTE]
> PayPay のような決済は、支払い完了の連絡が少し遅れて届くことがあります（Webhook 経由で届きます）。注文ステータスがすぐに「処理中」にならなくても、しばらく待つと自動的に更新されます。本プラグインは支払い連絡の到着まで注文を自動でキャンセルせずに待機し、必要に応じて PAY.JP へ確認しにいく仕組みを備えています。

## 5-4. 返金のテスト

注文画面からの返金も試しておきましょう。手順は [第7章 7-3. 返金する](07-daily-operations.md#7-3-返金する) と同じです。テスト決済の返金も同じ操作でできます。

## 5-5. テストで確認しておきたいことリスト

本番公開の前に、次の項目をひととおり確認しておくと安心です。

- [ ] クレジットカードで購入でき、注文が「処理中」になる
- [ ] わざと失敗するカード番号（`4000 0000 0000 0002`）でエラーが表示される
- [ ] PayPay で購入できる（100 円以下・終わったら全額返金）
- [ ] 注文画面から返金ができる
- [ ] 注文の確認メールが届く（WooCommerce の標準機能）

すべて確認できたら、いよいよ本番利用の申請に進みます。
