# 第6章 本番利用の申請と本番切り替え

テストで問題なく動くことを確認したら、実際のお金を受け取れるようにするための「本番申請」を行います。この章の流れは次のとおりです。

```
① PAY.JP に本番環境の利用を申請する（審査）
        ↓
② 審査に通ると本番用 API キーが使えるようになる
        ↓
③ プラグインに本番用キーを設定し、テストモードを解除する
        ↓
④ 本番環境の Webhook を登録する
        ↓
⑤ ★特別料率の申請フォームから申請する（忘れずに！）
```

## 6-1. 本番環境の利用を申請する

1. [PAY.JP ダッシュボード](https://pay.jp/) にログインします。
2. ダッシュボード内の「**本番環境の利用申請**」（アカウント設定内にあります）を開きます。
3. 画面の案内に従って、事業者情報・ショップ情報・売上の入金先となる銀行口座などを入力して申請します。

### 申請前にサイト側で準備しておくこと

審査では、実際に販売するサイトの内容も確認されます。次の点を整えておくとスムーズです。

- **特定商取引法に基づく表記**のページがある（事業者名、住所、連絡先、返品・キャンセルの条件など）
- **プライバシーポリシー**のページがある
- 販売する**商品・価格が実際にサイトに掲載**されている
- サイトが公開されており、審査担当者が閲覧できる状態になっている

> [!NOTE]
> 審査には通常、数営業日かかります。審査状況や結果は PAY.JP からのメールおよびダッシュボードで確認できます。審査内容や必要書類の詳細は PAY.JP のヘルプ・ガイドをご確認ください。

## 6-2. 審査通過後: 本番用 API キーを設定する

審査に通ると、PAY.JP ダッシュボードの「API」ページで**本番用のキー**（`pk_live_` / `sk_live_` から始まる文字列）が確認できるようになります。

1. WordPress 管理画面で「**WooCommerce**」→「**設定**」→「**PAY.JP**」タブを開きます。
2. 「**本番秘密鍵**」に `sk_live_` から始まるキーを入力します。
3. 「**本番公開鍵**」に `pk_live_` から始まるキーを入力します。
4. 「**テストモード**」の**チェックを外します**。
5. 「**変更を保存**」をクリックします。

> [!WARNING]
> テストモードのチェックを外した瞬間から、**実際のお金が動く本番決済**になります。切り替えは営業時間外など、落ち着いて作業できるタイミングで行うのがおすすめです。
> また、本番の秘密鍵（`sk_live_`）は絶対に外部へ漏らさないでください。

## 6-3. 本番環境の Webhook を登録する

テスト環境で行った Webhook の登録（第4章 4-7）を、**本番環境側でももう一度**行います。

1. PAY.JP ダッシュボードを**本番環境の表示**に切り替えます。
2. 「Webhook」設定ページで、プラグイン設定画面に表示されている Webhook URL（`https://あなたのサイト/wp-json/payjp/v2/webhook`）を登録します。
3. 本番環境の「Webhook Secret」をコピーし、プラグイン設定画面の「**Webhook シークレット**」欄に貼り付けて保存します（テスト用のシークレットから置き換えます）。

## 6-4. ★特別料率の申請（重要・忘れずに！）

> [!IMPORTANT]
> ## ここが一番大切なポイントです
>
> **PAY.JP の本番申請が完了しただけでは、特別料率は適用されません。**
>
> 本番申請の完了後に、**下記の専用フォームから特別料率の適用を申請**してください。
> この申請を行わない限り、決済手数料は**通常料率のまま**になります。申請が遅れた期間の手数料は原則さかのぼって適用されませんので、**本番申請が完了したらすぐに**申請することを強くおすすめします。
>
> ### 特別料率 申請フォーム
>
> **[特別料率申請フォームの URL をここに記載]**
>
> ### 申請の流れ
>
> 1. PAY.JP の本番環境の利用申請を完了させる（6-1）
> 2. 上記の申請フォームを開く
> 3. フォームの案内に従って、PAY.JP に登録した情報（アカウントのメールアドレス、ショップ名・サイト URL など）を入力して送信する
> 4. 適用の連絡を待つ（適用状況は PAY.JP からの案内をご確認ください）

### チェックリスト

本番公開前の最終確認です。

- [ ] 本番用 API キー（`pk_live_` / `sk_live_`）を設定した
- [ ] テストモードのチェックを外して保存した
- [ ] 本番環境の Webhook を登録し、本番用の Webhook シークレットを設定した
- [ ] **特別料率の申請フォームから申請した** ← 忘れていませんか？
- [ ] 少額の商品で本番決済を 1 件行い、正常に完了することを確認した（確認後に返金すれば OK です）

これで本番運用の準備は完了です。おつかれさまでした！
