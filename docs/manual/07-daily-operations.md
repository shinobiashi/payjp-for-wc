# 第7章 日々の運用

この章では、販売開始後の日常的な操作（注文の確認・返金・キャンセルなど）を説明します。

## 7-1. 注文と支払い状況を確認する

1. WordPress 管理画面で「**WooCommerce**」→「**注文**」を開きます。
2. 注文の一覧が表示されます。PAY.JP で支払われた注文のステータスは、通常次のようになります。

| ステータス | 意味 |
|-----------|------|
| **処理中** | 支払いが完了しています。商品の発送など、店舗側の対応を進めてください。 |
| **保留中** | お客様がまだ支払いを完了していない状態です（PayPay の支払い途中など）。支払いが完了すると自動で「処理中」に変わります。 |
| **完了** | 店舗側の対応（発送など）が終わった注文です。手動で変更します。 |
| **キャンセル** | キャンセルされた注文です。 |
| **返金済み** | 全額返金された注文です。 |

注文をクリックすると詳細画面が開き、画面右側の「注文メモ」欄に PAY.JP の決済に関する記録（支払い完了、返金、エラーなど）が自動で残ります。困ったときはまずここを確認してください。

## 7-2. 売上金の確認と入金

売上金額や入金予定は **PAY.JP ダッシュボード**で確認できます。入金のスケジュールや振込手数料は PAY.JP の規定に従います。詳しくは PAY.JP ダッシュボードおよびヘルプをご確認ください。

## 7-3. 返金する

返金は WooCommerce の注文画面から行えます。クレジットカード・PayPay のどちらも、全額返金・一部返金に対応しています。

1. 「WooCommerce」→「注文」から、返金したい注文を開きます。
2. 商品明細の下にある「**返金**」ボタンをクリックします。
3. 返金する金額を入力します。
   - 全額返金する場合は合計金額を入力します。
   - 一部だけ返金する場合はその金額を入力します。
4. 「**PAY.JP 経由で返金**」ボタンをクリックします。
5. 確認ダイアログで OK すると、PAY.JP 経由でお客様に返金されます。

> [!NOTE]
> 返金がお客様のカード明細や PayPay 残高に反映されるまでの日数は、カード会社や PayPay 側の処理によります。

## 7-4. 注文をキャンセルする

支払い済みの注文をキャンセルしたい場合は、注文詳細画面でステータスを「**キャンセル**」に変更して「更新」します。

- 支払いがまだ確定していない場合 → PAY.JP 側の決済も自動で中止されます。
- すでに支払いが確定している場合 → **自動で全額返金**されます。

キャンセル処理の結果は注文メモに記録されるので、確認してください。

## 7-5. 「与信のみ」（手動で売上を確定する）方式を使っている場合

第4章の「支払いキャプチャ」で「**与信のみ**」を選んでいる店舗向けの説明です（「即時キャプチャ（推奨）」のままの場合、この節は読み飛ばして構いません）。

- 「与信のみ」では、注文時点ではカード枠の確保（オーソリ）だけが行われ、まだ売上は確定していません。
- **注文のステータスを「完了」に変更すると、そのタイミングで売上が確定（キャプチャ）されます。** 商品の発送が済んだら、注文を「完了」にしてください。
- 売上を確定せずに注文を「キャンセル」にすると、確保していたカード枠が解放されます。

> [!WARNING]
> 与信（カード枠の確保）には有効期限があります。長期間そのままにすると自動的に失効しますので、発送が済んだら速やかに注文を「完了」にしてください。

## 7-6. お客様の保存カードについて

「支払い方法を保存する」を有効にしている場合、お客様は自分のアカウントページ（「マイアカウント」→「お支払い方法」）で保存したカードを確認・削除できます。カード番号そのものはあなたのサイトには保存されておらず、PAY.JP 側で安全に管理されています。

## 7-7. 異常があったときの通知メール

まれに「キャンセル済みの注文に対して、あとから支払い完了の連絡が届く」といったイレギュラーが起きることがあります。本プラグインはこうした異常を検知すると、**注文ステータスを勝手に変更せず**、注文メモへの記録と管理者へのメール通知を行います。

通知メールの宛先は「WooCommerce」→「設定」→「PAY.JP」タブの「**異常通知メールアドレス**」で変更できます（空欄の場合はサイト管理者のメールアドレスに届きます）。

通知が届いたら、注文メモと PAY.JP ダッシュボードの決済履歴を確認し、必要に応じて返金などの対応をしてください。
